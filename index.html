Your First Array

🔬 Scenario

A NumPy array is an ordered list of numbers you can do math on. Think of it as a single column from your plate reader export — but programmable.

viability = [98.2, 95.7, 97.1, 12.3, 11.8, 13.1] ↑ ↑ ↑ ↑ ↑ ↑ [0] [1] [2] [3] [4] [5] well1 well2 well3 well4 well5 well6 Index starts at 0, not 1. well1 = index 0, well6 = index 5.

import numpy as np

📅 Day 2 — Tuesday

Mean, SD, and %CV

⏱ 30–40 min 🆕 np.mean(), np.std(), if/else
🔬 Scenario
Your lab ran a CellTiter-Glo assay on MCF-7 breast cancer cells. You have triplicates for both your DMSO control and drug-treated group. Calculate QC statistics before reporting results.

💡 The Concept

%CV = (Standard Deviation / Mean) × 100

This is the core QC metric for every assay. Below 10% = acceptable for cell-based assays.

💻 Code — day2_statistics.py

import numpy as np
np.set_printoptions(suppress=True)

# MCF-7 breast cancer cells — CellTiter-Glo RLU values
dmso_control = np.array([45320.0, 44870.0, 45110.0])
drug_treated = np.array([8920.0,  9210.0,  8750.0])

dmso_mean = np.mean(dmso_control)
dmso_std  = np.std(dmso_control, ddof=1)  # ALWAYS ddof=1
dmso_cv   = (dmso_std / dmso_mean) * 100

drug_mean = np.mean(drug_treated)
viability = (drug_mean / dmso_mean) * 100

print(f"DMSO Mean: {dmso_mean:.1f}  SD: {dmso_std:.1f}  %CV: {dmso_cv:.2f}%")
print(f"Viability: {viability:.1f}%")

if dmso_cv < 10:
    print("QC Status: PASS")
else:
    print("QC Status: FAIL — replate recommended")

✅ Expected Output

DMSO Mean: 45100.0 SD: 226.5 %CV: 0.50% Viability: 19.9% QC Status: PASS
⚡ Mini Challenge

Change DMSO to [45320.0, 38200.0, 51100.0] — does QC now say FAIL?

📅 Day 5 — Monday

axis + for loops: Per-Row Statistics

⏱ 35–45 min 🆕 axis=1/0, for i in range(), elif
🔬 Scenario
You need a per-row summary of your plate — mean viability and a QC flag for each row. This is the standard format for plate QC reports in pharma.

💡 axis=1 vs axis=0

plate = [[96.0, 98.0, 94.0], row 0 [44.0, 46.0, 42.0], row 1 [ 4.0, 6.0, 2.0]] row 2 axis=1 → calculate ACROSS each row (→ → →) row 0 mean = (96+98+94)/3 = 96.0 ← one number per ROW row 1 mean = (44+46+42)/3 = 44.0 row 2 mean = ( 4+ 6+ 2)/3 = 4.0 result: [96.0, 44.0, 4.0] shape: (3,) axis=0 → calculate DOWN each column (↓ ↓ ↓) col 0 mean = (96+44+4)/3 = 48.0 ← one number per COLUMN result: [48.0, 50.0, 46.0] shape: (3,)

💡 The for Loop

for i in range(3): print(i) → i=0 → prints 0 → i=1 → prints 1 → i=2 → prints 2 range(3) generates: 0, 1, 2 (stops BEFORE 3)

📅 Day 6A — Wednesday

Boolean Masking: The Basics

⏱ 25–30 min 🆕 >, <, ==, array[ mask ]
🔬 Scenario
Your lab measured cell survival after drug treatment. You want to automatically flag wells below 20% — these are your potential drug hits.

💡 The Concept

wells = [98.2, 95.7, 12.3, 11.8, 97.1, 8.4] wells < 20: [False, False, True, True, False, True] ↑ ↑ ↑ only these wells pass the filter wells[mask]: [12.3, 11.8, 8.4] ← only True positions kept Step 1: Create mask mask = wells < 20 Step 2: Apply mask hits = wells[mask] Step 3: Use result print(hits)

💻 Code — day6a_masking.py

import numpy as np
np.set_printoptions(suppress=True)

wells = np.array([98.2, 95.7, 12.3, 11.8, 97.1, 8.4])

# Step 1: Create mask
mask = wells < 20
print("Mask:", mask)

# Step 2: Apply mask
hit_wells = wells[mask]
print("Hit wells:", hit_wells)

# Step 3: Count
print(f"Total: {wells.size}  |  Hits: {hit_wells.size}")

✅ Expected Output

Mask: [False False True True False True] Hit wells: [12.3 11.8 8.4] Total: 6 | Hits: 3

📅 Day 6B — Thursday

Combining Conditions: & and |

⏱ 30–35 min 🆕 & (AND), | (OR), np.sum()
🔬 Scenario
You want to flag drug hits that are below 30% viability AND not obvious measurement errors (which show up above 105%). One condition alone is not enough.

💡 The Concept

wells = [98.2, 12.3, 110.5, 8.4, 45.2, 3.1] condition 1: wells < 30 [False, True, False, True, False, True] condition 2: wells < 105 [ True, True, False, True, True, True] combined: (wells < 30) & (wells < 105) [False, True, False, True, False, True] ↑ ↑ ↑ BOTH conditions must be True to keep the well & = AND — both must be True | = OR — either can be True ⚠️ Always wrap each condition in parentheses! ✅ (wells < 30) & (wells < 105) ❌ wells < 30 & wells < 105 ← operator order bug

💻 Code — day6b_conditions.py

import numpy as np
np.set_printoptions(suppress=True)

wells = np.array([98.2, 12.3, 110.5, 8.4, 45.2, 3.1,
                  95.7, 22.1,  88.3, 0.2, 67.4, 14.8])

# AND: hits that are not errors
hit_mask  = (wells < 30) & (wells < 105)
hit_wells = wells[hit_mask]
print("Hits (< 30% AND < 105%):", hit_wells)

# OR: extreme wells (very high OR very low)
extreme_mask  = (wells > 100) | (wells < 5)
extreme_wells = wells[extreme_mask]
print("Extreme wells:", extreme_wells)

# Count Trues using np.sum()
print("Hit count:", np.sum(hit_mask))
# np.sum() on a boolean: True=1, False=0 → gives count

📅 Day 6C — Friday

np.abs() and ~ (NOT)

⏱ 25–30 min 🆕 np.abs(), ~ (invert mask)
🔬 Scenario
Some wells in your screen deviate far from the plate average — either too high or too low. You want to flag these outliers regardless of direction, then remove them to get clean data.

💡 np.abs() — Absolute Value

z_scores = [-2.8, 0.4, 1.2, -3.1, 0.9] np.abs(z_scores): [2.8, 0.4, 1.2, 3.1, 0.9] ↑ ↑ negative signs removed — now we can use > threshold to catch BOTH high and low outliers with one line

💡 ~ (NOT) — Inverting a Mask

mask = [False, False, True, True, False] ~mask = [ True, True, False, False, True] Use ~ to get "everything except": clean = wells[~outlier_mask] ← keep NON-outliers

💻 Code — day6c_abs_not.py

import numpy as np
np.set_printoptions(suppress=True)

deviations = np.array([-2.8, 0.4, 1.2, -3.1, 0.9, -0.3, 2.1, -1.7])

# Absolute value — removes negative signs
abs_dev = np.abs(deviations)
print("Absolute:", abs_dev)

# Flag outliers: |deviation| > 2.5
outlier_mask = np.abs(deviations) > 2.5
print("Outliers:", deviations[outlier_mask])

# Keep clean wells using ~
clean_wells = deviations[~outlier_mask]
print("Clean:", clean_wells)

print(f"Outliers: {np.sum(outlier_mask)}  |  Clean: {np.sum(~outlier_mask)}")

✅ Expected Output

Absolute: [2.8 0.4 1.2 3.1 0.9 0.3 2.1 1.7] Outliers: [-2.8 -3.1] Clean: [ 0.4 1.2 0.9 -0.3 2.1 -1.7] Outliers: 2 | Clean: 6
⚡ Mini Challenge

Find wells within normal range: np.abs(deviations) <= 1.0. How many?

Answer: [0.4, 0.9, -0.3, -1.7] → not quite, check the abs values carefully!
